There are multiple use cases:
- A story is Regular: the user wants to move it to Primary. The website prompts the user with a warning, telling them the current Primary story is ... and asking them if they want to move the current Primary to a Secondary position or Regular. If they select a Secondary position, the website should prompt them with another warning saying the current Secondary story for that slot will be moved to Regular.
The website after submitting should:
1) set the old primary to either regular or the new secondary, in which case the old secondary should move to regular (all entries false).
2) set the new story to primary.

- A story is Primary: the user wants to move it to Secondary. The website should prompt them with a warning saying the current Secondary story for that slot will be moved to Regular.
The website after submitting should:
1) set the old secondary to regular (all entries false).
2) set the new story to the new secondary slot.
3) warn the user there is currently no active primary.

- A story is Secondary:  the user wants to move it to Primary or Regular.  The website prompts the user with a warning, telling them the current Primary story is ... and asking them if they want to move the current Primary to a Secondary position other than the one the current story they want to move is on or Regular. If they select a Secondary position, the website should prompt them with another warning saying the current Secondary story for that slot will be moved to Regular.
The website after submitting should:
1) set the old primary to either regular or the new secondary, in which case the old secondary should move to regular (all entries false).
2) set the new story to primary or regular.
3) the website should warn the user there is an empty slot for secondary.

The website should not show warnings if a story is not moved spot, and the website should not allow changes within a category (ie, secondary 1 to secondary 3).
